# LIFE·LAB 操作説明書

Conway のライフゲーム研究用シミュレーター **LIFE·LAB** の完全な操作マニュアルです。
盤面は端で折り返さない**無限グリッド**として実装されており、グライダーや銃が生成するパターンは壁にぶつからず無限遠へ飛んでいきます。

## 1. 画面構成

画面は大きく3つの領域に分かれています。

**上部バー（トップバー）**
再生／一時停止、1世代送り、速度スライダー、世代数・人口カウンター、全消去、ヘルプボタンが並びます。

**左サイドパネル**
ツール選択、パターンライブラリ、ルール設定、表示オプション、ランダムスープ、RLE 入出力をまとめた作業パネルです。左上の三本線アイコン（≡）で開閉できます。**閉じるとキャンバスが自動的に広がり、隠れていた盤面が見えるようになります。**

**中央キャンバス**
セルオートマトンが実際に動く主画面です。四隅・周辺には次の補助表示があります。

- **左下 HUD** — カーソル座標／パターンの外接矩形サイズ／ズーム倍率／実行速度
- **上中央のバッジ** — 操作の確認メッセージ（数秒で消えます）
- **右下** — ズームイン（＋）、ズームアウト（−）、フィット（Fit）
- **原点の十字マーク** — 座標 (0, 0) を示す琥珀色の小さな十字

## 2. 基本操作：セルを描く・消す

セルの編集は左パネル上部の **Tools** から行います。

| ツール | ショートカット | 動作 |
|--------|:---:|------|
| **Draw**（描画） | `B` | クリック／ドラッグした位置のセルを**生**にします |
| **Erase**（消去） | `E` | クリック／ドラッグした位置のセルを**死**にします |

- Draw と Erase は「トグル（反転）」ではなく、**塗り**として動作します。Draw は常に生セルを置き、Erase は常に消します。
- ドラッグ中は経路上のセルが連続して塗られ、1ストロークの間に同じセルを二度処理しないよう制御されているため、素早くドラッグしても意図通りに描けます。
- 1ストローク（マウスを押してから離すまで）ごとに自動でアンドゥ地点が記録されます。

> **セル単位の厳密な配置**
> 十分にズームインした状態（グリッド線が見える倍率）でクリックすれば、狙った1セルだけを正確に打てます。初期状態を厳密に設定したいときは、まず `＋` またはホイールで拡大してから描画してください。

## 3. シミュレーションの再生と速度

**再生／一時停止**
トップバー左端の **Run / Pause** ボタン、または `Space` キーで切り替えます。再生中はボタンが一時停止アイコンに変わります。

**1世代だけ進める（ステップ実行）**
Run の隣のステップボタン、または `S` / `→` キーで1世代だけ進めます。再生中に押すと自動的に一時停止してから1世代進むので、パターンをコマ送りで観察できます。

**速度スライダー**
トップバーの **Speed** スライダーで実行速度を調整します。内部では次のようにマッピングされています。

- **低速側**（スライダー左寄り）：1世代ずつを、600ミリ秒ごと〜16ミリ秒ごとの間隔で実行。振動子や小さなパターンをゆっくり観察するのに向きます。
- **高速側**（スライダー右寄り）：毎フレーム（約60fps）で **1〜48世代** を一気に計算。メスセラ（長寿命パターン）の遠い未来まで一気に探索できます。

現在の実効速度は HUD の **Speed** 欄（例：`8 gen/s`）とスライダー横のラベルに表示されます。

## 4. 視点操作：パン・ズーム・フィット

無限グリッドを自在に見回すための操作です。

**パン（視点移動）**
次のいずれかの方法で盤面をドラッグ移動できます。

- **Pan ツール**（`H`）を選んでドラッグ
- **`Space` を押しながら**ドラッグ（どのツール中でも一時的にパンできます）
- **マウス中ボタン**でドラッグ（どのツール中でも常に有効）
- 矢印キー `←` `↑` `↓` でも少しずつ移動できます（`→` はステップ実行に割り当てられているため移動には使いません）

**ズーム**
- **マウスホイール**：カーソル位置を中心に拡大／縮小します。注目したい部分にカーソルを合わせて回すのがコツです。
- 右下の **＋ / −** ボタン、またはキーボードの `+` / `-`：画面中央を中心にズームします。
- 倍率はおよそ **0.08倍〜64倍** の範囲です。グリッド線は4倍以上で表示されます。

**フィット**
右下の **Fit** ボタン、または `F` キーで、現在盤上にある全パターンが画面に収まるよう自動的にズーム・位置合わせします。パターンを見失ったときの復帰にも便利です。

## 5. パターンライブラリ（スタンプ）

有名なパターンをワンクリックで盤面に配置できます。左パネルの **Pattern Library** から選びます。

**カテゴリと収録パターン**

| カテゴリ | パターン |
|----------|----------|
| Quadratic growth（二次成長） | Max、Spacefiller、Breeder 1 |
| Spaceships（宇宙船） | Glider、Lightweight ship (LWSS) |
| Oscillators（振動子） | Blinker、Toad、Beacon、Pulsar、Pentadecathlon |
| Guns（銃） | Gosper glider gun |
| Methuselahs（メスセラ） | R-pentomino、Acorn、Diehard |
| Still lifes（静物） | Block、Beehive、Loaf、Boat、Tub |

各行の右側には周期や寿命などのメタ情報（例：Pulsar の `p3`、R-pentomino の `1103 gen`）が表示されます。

**Quadratic growth（二次成長）カテゴリ**

生存セル数が世代 $t$ の2乗に比例して増え続ける、ライフゲームで理論上最速の成長を示すパターン群です。

- **Max**（187セル・27×27／Tim Coe, 1995）— 最小級のスペースフィラー。平面をゼブラ模様の静物で埋め尽くします。
- **Spacefiller**（200セル・49×26／Hensel・Holzwart・Bell・Coe）— 古典的なスペースフィラー。四方向すべてに c/2（理論上の最大速度）で、密度 1/2（理論上の最大密度）まで空間を充填します。
- **Breeder 1**（4060セル・749×338／Bill Gosper, 1970年代初頭）— 二次成長を示すパターンとして史上初めて発見されたもの。グライダー銃を線形に生み出し、各銃がグライダーを撒くことで全体として $O(t^2)$ の成長になります。

Breeder 1 のような大きなパターンを選ぶと、**スタンプ全体が画面に収まるよう自動的にズームアウト**されます（`Ctrl+V` での貼り付け時も同様）。二次成長は Breeder 1 では数千世代かけて現れるため、速度スライダーを高速側にして長く走らせてください。Max と Spacefiller は数十世代で成長が見て取れます。

**配置の手順**

1. ライブラリからパターン名をクリックして選択します（選択中は行がハイライトされます）。
2. カーソルをキャンバス上に動かすと、配置プレビューが**ティール（青緑）色**で表示されます。
3. 置きたい位置でクリックすると確定します。
4. **`R` キーで90°回転**、**`Esc` キーで選択解除**できます。連続して何個でも置けます。

## 6. 矩形選択ツール

盤面の一部を範囲指定して操作します。**Select ツール**（`M`）を選び、キャンバス上をドラッグして矩形範囲を作ります。範囲を確定すると、範囲の近くに小さなフローティングバーが現れます。

| ボタン | 動作 |
|--------|------|
| **Copy RLE** | 選択範囲内のセルを RLE 文字列に変換し、RLE 入出力欄とクリップボードにコピー |
| **Random** | 選択範囲を、密度スライダーの値に従ってランダムなセルで埋め直す |
| **Clear** | 選択範囲内のセルをすべて消去 |

`Esc` キーで選択を解除できます。パターンの一部を切り出して保存したり、特定領域だけをリセットするのに使います。

**コピー&ペースト(`Ctrl+C` / `Ctrl+X` / `Ctrl+V`)**

選択範囲がある状態で `Ctrl+C`(Mac は `⌘+C`)を押すと、範囲内の**生きたセル**が内部クリップボードにコピーされます(同時に RLE 文字列としてシステムのクリップボードにも書き出されます)。`Ctrl+X` はコピーした上で範囲内のセルを消去します。選択バーの **Copy RLE** ボタンでも同様に内部クリップボードへ入ります。

`Ctrl+V` を押すと、コピーした配置がスタンプとしてカーソルに追従します。好きな場所をクリックすれば**何個でも連続配置**でき、ライブラリのスタンプと同じく `R` で90°回転、`Esc` で解除です。

- **シミュレーション実行中でも**コピー・貼り付けの両方が機能します。走行中の盤面から動いている構造(グライダー銃、宇宙船など)を捕まえて、別の場所へ「乱入」させる遊び方・実験がそのままできます。
- 貼り付けは既存セルへの**追加**(OR 合成)です。貼り付け先のセルを消すことはありません。
- 外部サイト(LifeWiki 等)でコピーした RLE テキストやプレーンテキスト配置も、キャンバス上で `Ctrl+V` すればそのままスタンプになります。`□`/`■` で書かれたグリッドにも対応しています。

## 7. ランダムスープ

初期パターンの「種」をランダムに撒く機能です。左パネルの **Random Soup** から操作します。

1. **Density（密度）** スライダーで生セルの割合を 5%〜70% の範囲で設定します。
2. **Fill viewport with soup** ボタンを押すと、**現在画面に表示されている範囲全体**がランダムなセルで埋められます。

ランダムスープを流して安定するまで走らせると、静物や振動子が自然発生します。未知のパターン探しの出発点として使ってください。撒く範囲は「今見えている領域」なので、事前にズーム倍率と位置を調整しておくと撒く広さをコントロールできます。

## 8. RLE 入出力

**RLE (Run Length Encoded)** は Golly や LifeWiki で標準的に使われるパターン記述形式です。ここで作ったものを保存・共有したり、外部のパターンを読み込んだりできます。左パネル最下部の **RLE Import / Export** で操作します。

**読み込み（Load）**
テキスト欄に RLE を貼り付けて **Load** を押すと、盤面がクリアされてパターンが読み込まれ、画面中央に配置された上で自動的にフィットされます。

- 標準的な RLE ヘッダ（`x = …, y = …, rule = …`）とコメント行（`#` 始まり）に対応しています。
- ヘッダにルールが書かれていれば、それも自動で読み込まれます。
- RLE 本体が無い場合は、`.`（死）と `O`／`*`／`X`（生）、または `□`／`■` で書かれた**プレーンテキスト形式**としても解釈を試みます。

**書き出し（Export board）**
**Export board** を押すと、現在盤上の全セルが RLE 文字列に変換されてテキスト欄に出力されます。現在のルールがヘッダに埋め込まれます。

**コピー（Copy）**
テキスト欄の内容をクリップボードにコピーします。

> 出力される RLE は Golly / LifeWiki 互換なので、LIFE·LAB で発見したパターンをそのまま他ツールで開いたり、掲示板等で共有したりできます。

## 9. ルール編集（他のセルオートマトン）

LIFE·LAB のエンジンは Conway のライフゲームに限らず、**Life-like セルオートマトン**全般を同じ仕組みで実行できます。左パネルの **Rule** 欄に `B/S` 記法で入力します。

**記法**
`B<誕生する近傍数>/S<生存する近傍数>` の形式です。数字は 0〜8。

- `B3/S23` — Conway のライフゲーム（既定値）
- `B36/S23` — **HighLife**（自己複製する複製子で有名）
- `B2/S` — **Seeds**（全セルが毎世代必ず死ぬ爆発的なルール）

入力後にフォーカスを外す（Enter または他をクリック）と適用されます。書式が不正な場合は入力欄が赤くなります。

## 10. 表示オプション

左パネルの **Display** で見た目を切り替えます。

**Age colouring（年齢カラーリング）**
セルを「生まれてからの経過世代」で色分けします。新しく生まれたセルは**琥珀色**、長く生き続けているセルは約22世代かけて**スレート（青灰色）**へと変化します。この結果、

- 活発に動いている領域・振動している領域 → 明るく光る
- 変化しない静物 → 静かに沈んだ色になる

ため、パターンのどこが「生きて動いている」かが一目で分かります。研究時にオシレータや wick、安定領域を見分けるのに有効です。オフにすると全セルが単色（ボーン色）になります。

**Grid lines（グリッド線）**
セルの格子線の表示・非表示を切り替えます（4倍以上のズームで表示）。10セルごとの線はやや濃く描かれ、目盛りの役割を果たします。

## 11. 取り消し（アンドゥ）

`Ctrl + Z`（Mac は `⌘ + Z`）で直前の操作を取り消せます。最大40ステップまで遡れます。

アンドゥ地点は次のタイミングで自動記録されます：描画・消去の各ストローク、パターンのスタンプ、ランダムスープ、選択範囲への操作、全消去、そして**再生を開始する直前**。再生前の状態がひとつ記録されるため、走らせてしまったパターンを再生前の初期状態へ一手で戻せます。

## 13. 座標系について

- 原点 **(0, 0)** は琥珀色の十字マークで示されます。
- X 軸は右方向が正、Y 軸は**下方向が正**です（画面座標に準じます）。
- カーソル位置の座標は HUD の **Cursor** 欄にリアルタイム表示されます。
- 盤面は無限グリッドですが、座標の有効範囲は各方向おおよそ **±3,300万セル** です。手動で走らせる範囲では事実上無制限で、トーラス（端の折り返し）による干渉は起こりません。

## 14. 活用のヒント

**振動子の周期を数える**
Age colouring をオンにし、速度を低速にして `S` でコマ送りすると、色の変化から周期が視覚的に追えます。HUD の Bounds（外接矩形）が周期的に元へ戻ることも手がかりになります。

**メスセラの終端を探る**
R-pentomino や Acorn をスタンプし、速度を高速側（毎フレーム多世代）にして一気に進めます。人口カウンターが安定したら決着（静物＋振動子＋飛び去ったグライダー）に達しています。途中で `F` を押すと拡散した破片全体を見渡せます。

**新パターンの発見と保存**
ランダムスープを撒いて安定させ、面白い部分を **Select → Copy RLE** で切り出せば、そのパターンだけを RLE として保存できます。得られた RLE は他のライフゲームツールでも開けます。

**別ルールの探索**
同じ初期配置でも `B3/S23` と `B36/S23`（HighLife）では挙動が大きく変わります。ルールを変えながら同じスープを走らせて比較すると、ルール空間の性質の違いが体感できます。
